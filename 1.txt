ЗАДАЧА: Реализовать автозагрузку store.js при первом запуске Lampa, добавить иконку магазина возле поиска в главном меню, добавить пункт в раздел Настроек, и обеспечить полностью корректную работу управления через пульт ТВ и мышь на ПК — используя только официальные API Lampa.

КОНТЕКСТ (по документации yumata/lampa-source):
Официальные точки расширения Lampa для плагинов:
- Lampa.Component.add() — регистрация новых экранов/активностей
- Lampa.Menu.addButton() — добавление кнопок/иконок в главное меню (там же где иконка поиска)
- Lampa.SettingsApi.addComponent() — добавление раздела в Настройки
- Lampa.Plugins.push() — программная инъекция плагина в текущую сессию (то, что должно решить проблему "плагины не появляются в Расширениях")
- Lampa.Storage — персистентное хранение (extensions — список активных расширений, ключ для автозагрузки нужно исследовать/использовать тот же паттерн)
- Lampa.Controller / .selector — навигация пультом (уже обсуждали в прошлой итерации, применить те же принципы здесь)

===== ЧАСТЬ 1: АВТОЗАГРУЗКА STORE.JS ПРИ ПЕРВОМ ЗАПУСКЕ =====

Цель: пользователь один раз вручную добавляет ссылку на bootstrap-скрипт через Настройки → Расширения → Добавить плагин, а дальше store.js должен сам проверять обновления и подтягивать актуальную версию с GitHub при каждом запуске Lampa, без повторного ручного вмешательства.

Реализовать:
1. При инициализации плагина проверить Lampa.Storage.get('urrowstore_first_run', true).
2. Если true — выполнить первичную настройку: зарегистрировать плагин в Lampa.Plugins.push({...}) программно (согласно документации, это официальный способ инъекции плагина в активную сессию, а не самодельная запись в произвольный storage-ключ, как было раньше — это исправляет прошлый баг с "плагин не виден в Расширениях").
3. Установить Lampa.Storage.set('urrowstore_first_run', false) после успешной инициализации.
4. При каждом последующем запуске — выполнить проверку версии: fetch('https://raw.githubusercontent.com/k1run9/urrowstore/main/version.json') (создать этот файл в репозитории с полем {"version": "x.y.z"}), сравнить с локально сохранённой версией в Lampa.Storage.get('urrowstore_version').
5. Если версия на GitHub новее — подтянуть актуальный store.js через динамическую вставку <script src="https://k1run9.github.io/urrowstore/store.js?v=НОВАЯ_ВЕРСИЯ"> (versioned query param решает проблему кэширования), обновить сохранённую версию в Storage.
6. Обернуть весь процесс проверки в try/catch с таймаутом на fetch (5 секунд), чтобы отсутствие интернета не блокировало запуск Lampa — при ошибке использовать локально закэшированную версию плагина без обновления.

===== ЧАСТЬ 2: ИКОНКА ВОЗЛЕ ПОИСКА В ГЛАВНОМ МЕНЮ =====

Использовать официальный Lampa.Menu.addButton() вместо самодельного addHeaderButton/addIcon, который вызывал ошибки в прошлых итерациях:
1. Найти актуальную сигнатуру Lampa.Menu.addButton() (исследовать параметры: обычно принимает объект с полями title, icon (SVG-строка), action (callback) — подтвердить через тестовый вызов с логированием).
2. Добавить кнопку с понятной SVG-иконкой (например значок магазина/плейлиста), которая по клику/нажатию ОК открывает главный экран магазина плагинов через Lampa.Activity.push({component: 'urrowstore_main', ...}).
3. Кнопка должна физически располагаться в основном меню навигации рядом с поиском — проверить порядок вставки через Menu.addButton (обычно добавляется в конец списка, но нужно протестировать визуальное размещение и при необходимости использовать параметр позиции, если API это поддерживает).
4. Кнопка автоматически получает поддержку пульта, так как использует официальный Menu API — Lampa сама применяет .selector и Controller-логику к элементам меню (это должно устранить весь класс проблем с навигацией, которые были при самодельной реализации).

===== ЧАСТЬ 3: РАЗДЕЛ В НАСТРОЙКАХ =====

Использовать Lampa.SettingsApi.addComponent():
1. Зарегистрировать новый компонент настроек, например:
   Lampa.SettingsApi.addComponent({
       component: 'urrowstore_settings',
       icon: '<svg>...</svg>',
       name: 'Магазин плагинов'
   });
2. Добавить внутрь параметры через Lampa.SettingsApi.addParam(): переключатель "Автообновление магазина" (boolean), кнопка "Проверить обновления сейчас" (действие), информация о текущей установленной версии store.js.
3. Убедиться что раздел появляется в общем списке Настроек Lampa и корректно навигируется пультом — так как SettingsApi это официальный компонент, стандартная навигация Lampa должна работать "из коробки" без дополнительного кода Controller (в отличие от кастомных экранов).

===== ЧАСТЬ 4: ГЛАВНЫЙ ЭКРАН МАГАЗИНА — ПОЛНАЯ ПОДДЕРЖКА ПУЛЬТА И МЫШИ =====

Зарегистрировать главный экран через Lampa.Component.add('urrowstore_main', function(object){...}) — это гарантирует, что Lampa обрабатывает его как полноценную Activity со встроенной поддержкой Controller, а не как самодельный оверлей (частая причина поломки навигации в прошлых версиях).

Внутри компонента:
1. this.create() — метод обязателен (устраняет прошлую ошибку "this.component.create is not a function"), должен вернуть jQuery-объект с разметкой, где каждая карточка плагина имеет класс .selector.
2. this.start() — вызывается Activity после создания, здесь регистрировать Controller через Lampa.Controller.add с move/enter/back методами, привязанными к this.activity/this.render() (стандартный паттерн Lampa-компонентов).
3. this.back() — обрабатывает нажатие "Назад" на пульте, должен вызывать Lampa.Activity.backward().
4. this.destroy() — обязателен для очистки слушателей/таймеров при закрытии экрана (защита от утечек памяти при частом открытии/закрытии).
5. Для мыши на ПК: элементы с .selector автоматически кликабельны через встроенный Lampa hover/click handler — никакой дополнительной логики не требуется, если структура компонента соответствует стандарту Lampa.Component (проверить, что клик мышью по карточке вызывает тот же handler, что и Enter на пульте — обычно достигается через единый onClick метод, привязанный и к клику и к Controller.enter()).

===== ЧАСТЬ 5: УСТАНОВКА ПЛАГИНОВ — РЕАЛЬНАЯ ИНТЕГРАЦИЯ =====

При клике "Установить" на карточке плагина в каталоге:
1. Вызвать Lampa.Plugins.push({url: script_url, name: plugin_name}) — официальный метод программной установки, согласно документации это тот же путь, которым Extension.js устанавливает плагины вручную добавленные пользователем.
2. Дополнительно продублировать в Lampa.Storage под тем же ключом, что использует нативный раздел Расширения (исследовать точное имя ключа через анализ Extension.js — согласно документации, вероятно 'plugins' или синхронизируется через Account.Api, если пользователь залогинен в CUB).
3. После установки — вызвать принудительное обновление UI раздела расширений, если он открыт (Lampa.Settings.update() или аналогичный метод перерисовки).
4. Показать Lampa.Noty('Плагин установлен: ' + plugin_name) как подтверждение пользователю.
